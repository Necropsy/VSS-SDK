Utilização e Desenvolvimento

Última Atualização: 21 de Nobembro de 2016

Utilização Básica

Existem uma série de tags que podem ser empregadas na execução do VSS-SDK pelo shellscript run.sh, vale lembrar que esse método de execução é destinado somente a um computador. As tags são: vision, sim, viewer, joy, sample, debug, develop e fast.

vision

Adiciona o VSS-Vision a lista de execução. Incompatível com a execução do VSS-Simulator.

sim

Adiciona o VSS-Simulator a lista de execução. Incompatível com a execução do VSS-Vision.

viewer

Adiciona o VSS-Viewer a lista de execução.

joy

Adiciona o VSS-Joystick a lista de execução, como estratégia do time amarelo.

sample

Adiciona um VSS-SampleCpp [branch:master] a lista de execução.

debug

Ativa o sistema de debug visual no VSS-Viewer, isto é, o desenho de poses finais, vetores de movimento e caminhos.

develop

Ativa o modo de desenvolvimento no VSS-Simulator, isto é, não há mais limite de tempo ou de gols para uma partida finalizar.

fast

Ativa o modo de execução acelerada no VSS-Simulator, isto é, o tempo passa 15 vezes mais rápido.


Alguns exemplos de como as tags podem ser empregadas na execução:

Simulação acelerada sem interface gráfica

./run.sh vision viewer

Nesse modo, o VSS-Vision irá rastrear objetos em campo e o VSS-Viewer irá desenhar os objetos em campo como o ambiente real.

Simulação acelerada sem interface gráfica

./run.sh sim fast yellow=sample blue=sample

Nesse modo, a simulação de um jogo de 10 minutos ocorrerá em cerca de 40 segundos sem ambiente gráfico, o VSS-Simulator irá plotar algumas informações do jogo no terminal e ao final irá retornar o relatório de jogo.

Simulação com interface gráfica

./run.sh sim viewer yellow=sample blue=sample

Nesse modo, ocorrerá a simulação de uma partida em 10 minutos. O jogo poderá ser assistido no VSS-Viewer e o VSS-Simulator continuará a plotar informações de jogo e retornar o relatório final de jogo.

Simulação com interface gráfica e informações de debug

./run.sh sim viewer debug yellow=sample blue=sample

Nesse modo, ocorrerá a simulação de uma partida de 10 minutos com debug visual. O jogo poderá ser assistido no VSS-Viewer, que além de desenhar o mundo também irá desenhar poses finais, vetores de movimento e caminhos.

Controlando um robô no simulador com um joystick usb

./run.sh sim viewer yellow=joy

Nesse modo, ocorrerá a simulação de uma partida de 10 minutos, onde um jogador irá controlar um robô amarelo. O jogo poderá ser assistido no VSS-Viewer como um videogame.

Modo de desenvolvimento com informações de debug

./run.sh sim viewer debug develop

Nesse modo, a simulação não termina até que o processo seja fechado. O VSS-Simulator e o VSS-Viewer estarão a espera de uma estratégia. Note que, você pode executar uma estratégia por fora do run.sh, verificar um erro, fechar a estratégia, modificá-la e testar novamente.

Utilização Avançada

Para que seja possível executar o projeto de maneira mais maleável e em diferentes máquinas é necessário deixar um terminal para cada aplicação. Esse tipo de execução é bastante comum em sistemas desacoplados e quase que padrão em grandes projetos de robótica. Todos os projetos possuem parâmetros de execução e para vê-los basta entrar com o parâmetro -h.

Parâmetros do VSS-Simulator

./VSS-Simulator -h
-h [ --help ] (Opcional) Exibe todos os parâmetros possiveis.
-f [ --fast ] (Opcional) Aumenta a velocidade do decorrer do tempo em 15 vezes.
-d [ --develop ] (Opcional) Ativa o modo de desenvolvimento.
-g [ --goal ] (Opcional) Aumenta a quantidade de gols necessários para acabar uma partida. Valores entre 10 e 100.


Parâmetros do VSS-Viewer

./VSS-Viewer -h
-h [ --help ] (Opcional) Exibe todos os parâmetros possiveis.
-c [ --camera ] (Opcional) Muda a posição da camera (arg=tv ou top).
-d [ --debug ] (Opcional) Ativa a recepção e desenho do debug de caminhos e poses.
-i [ --ip_state ] (Opcional) Define o IP de uma máquina na rede que esteja rodando o VSS-Vision ou VSS-Simulator.


Parâmetros VSS-Joystick

./VSS-Joystick -h
-h [ --help ] (Opcional) Exibe todos os parâmetros possiveis.
-i [ --ip_sender ] (Opcional) Define o IP de uma máquina na rede que esteja rodando o VSS-Simulator
-r [ --real ] (Opcional) Envia comando para robôs reais ao invés do VSS-Simulator. Porém é necessário adicionar o módulo de comunicação.


Parâmetros VSS-SampleCpp [branch:master]

./VSS-SampleStrategy -h
-h [ --help ] (Opcional) Exibe todos os parâmetros possiveis.
-d [ --debug ] (Opcional) Ativa a envio de debug de caminhos e poses.
-n [ --name ] (Opcional) Define um nome para a estratégia.
-i [ --ip_receive_state ] (Opcional) Define o IP de uma máquina na rede que esteja rodando o VSS-Vision ou VSS-Simulator.
-I [ --ip_send_debug ] (Opcional) Define o IP de uma máquina na rede que esteja rodando o VSS-Viewer
-s [ --ip_send_command ] (Opcional) Define o IP de uma máquina na rede que esteja rodando o VSS-Simulator.
-c [ --color ] (Obrigatório) Define a cor do tipo (arg=yellow ou blue)

Desenvolvimento

Utilizamos sockets para possibilitar a comunicação entre os projetos em uma máquina ou em várias. É utilizado a biblioteca protobuf para serializar os dados e a biblioteca ZMQ para o envio via sockets. Com essa combinação é possível desenvolver uma estratégia (inteligência) em qualquer linguagem de programação que suporte protobuf, como: C++, Rust, Java, Python, JavaScript, Go, Ruby, Objective-C, C#, Perl, PHP, Scala e Julia.
